The Avo Tracking Plan is where you define the data for your application. It contains
detailed definitions of your tracking plan that can be used to generate
code from and validate your data against.

The following sections build up your Avo workspace:

- <Link passHref href="/workspace/tracking-plan/events">
    <a>Tracking Plan Events</a>
  </Link>
  : where all events structures are defined
- <Link passHref href="/workspace/tracking-plan/properties">
    <a>Tracking Plan Properties</a>
  </Link>
  : where all properties are defined
- <Link passHref href="/workspace/tracking-plan/metrics">
    <a>Tracking Plan Metrics</a>
  </Link>
  : where all metrics are defined
- <Link passHref href="/workspace/tracking-plan/publishing">
    <a>Tracking Plan Publishing</a>
  </Link>
  : sync your tracking plan with the one in your analytics platform
- <Link passHref href="/workspace/connections">
    <a>Connection Setup</a>
  </Link>
  : configure sources and destinations, including setting up Inspector and Avo Codegen
- <Link passHref href="/workspace/implement">
    <a>Codegen Reference</a>
  </Link>
  : source specific event reference for Avo Codegen, with code snippets that help
  you implement
- <Link passHref href="/workspace/inspector">
    <a>Inspector</a>
  </Link>
  : get a report of the state of your tracking and get alerted when new issues are
  detected
